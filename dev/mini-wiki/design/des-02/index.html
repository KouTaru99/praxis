Chặng 2 · Thiết kế & ADR

Thiết kế schema database (Prisma)

Mục tiêu

Biến ERD thành schema Prisma thật + file migration SQL + biến môi trường mẫu.

Sản phẩm của bước: api/prisma/schema.prisma, migration 0001_init, và api/.env.example.

Sau bước này bạn sẽ thấy: Bạn sẽ có một schema database chính thức, type-safe, sẵn sàng để Chặng 3 áp vào PostgreSQL thật.

Hướng dẫn

ERD ở Chặng 1 là bản vẽ. Giờ ta biến nó thành schema Prisma — một file .prisma mô tả toàn bộ database bằng cú pháp ngắn gọn, từ đó Prisma tự sinh TypeScript client (type-safe) và file migration SQL. Lưu ý cách Prisma ánh xạ: tên model viết hoa kiểu Article nhưng @@map("articles") trỏ về tên bảng thật chữ thường. Trường publishedAt ánh xạ cột published_at qua @map.

Tạo api/prisma/schema.prisma:

api/prisma/schema.prisma
// Prisma Schema — Mini-Wiki v1
// Chạy `prisma generate` sau mỗi lần thay đổi file này.

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// ─── Article (Bài viết) ───────────────────────────────
model Article {
  id          Int          @id @default(autoincrement())
  title       String       @db.VarChar(255)
  slug        String       @unique @db.VarChar(255)
  content     String
  publishedAt DateTime?    @map("published_at") @db.Timestamptz()
  createdAt   DateTime     @default(now()) @map("created_at") @db.Timestamptz()
  updatedAt   DateTime     @updatedAt @map("updated_at") @db.Timestamptz()
  tags        ArticleTag[]

  @@map("articles")
}

// ─── Tag (Nhãn) ───────────────────────────────────────
model Tag {
  id        Int          @id @default(autoincrement())
  name      String       @unique @db.VarChar(100)
  slug      String       @unique @db.VarChar(100)
  createdAt DateTime     @default(now()) @map("created_at") @db.Timestamptz()
  articles  ArticleTag[]

  @@map("tags")
}

// ─── ArticleTag (Bảng nối nhiều-nhiều) ────────────────
model ArticleTag {
  articleId Int     @map("article_id")
  tagId     Int     @map("tag_id")
  article   Article @relation(fields: [articleId], references: [id], onDelete: Cascade)
  tag       Tag     @relation(fields: [tagId], references: [id], onDelete: Restrict)

  @@id([articleId, tagId])
  @@map("article_tags")
}

Tạo file migration SQL api/prisma/migrations/0001_init/migration.sql — đây là lệnh SQL thật sẽ chạy lên PostgreSQL ở Chặng 3 (chú ý GIN index cho Full-Text Search ở cuối):

api/prisma/migrations/0001_init/migration.sql
-- Migration 0001_init — Mini-Wiki v1

CREATE TABLE articles (
  id           SERIAL        PRIMARY KEY,
  title        VARCHAR(255)  NOT NULL,
  slug         VARCHAR(255)  NOT NULL UNIQUE,
  content      TEXT          NOT NULL,
  published_at TIMESTAMPTZ,  -- NULL = bản nháp (khớp schema.prisma DateTime?, KHÔNG default)
  created_at   TIMESTAMPTZ   DEFAULT NOW(),
  updated_at   TIMESTAMPTZ   DEFAULT NOW()
);

CREATE TABLE tags (
  id         SERIAL       PRIMARY KEY,
  name       VARCHAR(100) NOT NULL UNIQUE,
  slug       VARCHAR(100) NOT NULL UNIQUE,
  created_at TIMESTAMPTZ  DEFAULT NOW()
);

CREATE TABLE article_tags (
  article_id INT NOT NULL REFERENCES articles(id) ON DELETE CASCADE,
  tag_id     INT NOT NULL REFERENCES tags(id)     ON DELETE RESTRICT,
  PRIMARY KEY (article_id, tag_id)
);

-- Index Full-Text Search (GIN) trên title + content, dictionary 'simple'
CREATE INDEX idx_articles_fts
  ON articles
  USING GIN (to_tsvector('simple', title || ' ' || content));

Cuối cùng, tạo api/.env.example — file mẫu khai báo biến môi trường. File .env thật (có mật khẩu) đã bị .gitignore chặn; ta chỉ commit file .example để người khác biết cần điền gì:

api/.env.example
# Sao chép thành .env và điền giá trị thật:  cp .env.example .env
# File .env KHÔNG commit (đã có trong .gitignore).

# Chuỗi kết nối PostgreSQL
DATABASE_URL="postgresql://dev:dev@localhost:5432/miniwiki?schema=public"

# Origin được phép gọi API (CORS) — trỏ tới front-end React/Vite
CORS_ORIGIN="http://localhost:5173"

# Port Express server lắng nghe
PORT=3000
Cạm bẫy: Một bẫy tinh vi: published_at phải nullable, KHÔNG đặt DEFAULT NOW(). Trong schema, ta khai báo publishedAt DateTime? (dấu ? = cho phép NULL). Nếu migration để DEFAULT NOW(), mọi bài tạo ra sẽ tự "đã đăng" ngay — phá vỡ tính năng bản nháp (FR-3: bài nháp có published_at = NULL). Schema và migration phải khớp nhau, nếu lệch sẽ sinh bug khó tìm.

git add api/prisma api/.env.example
