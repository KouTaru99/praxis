Chặng 3 · Backend + Cơ sở dữ liệu

CRUD Tag & bảo vệ xóa

Mục tiêu

Viết API quản lý Tag, với ràng buộc không cho xóa tag đang gắn bài viết.

Sản phẩm của bước: api/src/controllers/tags.controller.ts + routes; 3 endpoint tag hoạt động, FR-9 được enforce.

Sau bước này bạn sẽ thấy: Bạn sẽ tạo được tag mới, và khi thử xóa tag đang dùng sẽ nhận lỗi 409 rõ ràng thay vì lỗi DB khó hiểu.

Hướng dẫn

Tag đơn giản hơn Article (không có Markdown, không slug tiếng Việt phức tạp). Điểm đáng học nằm ở xóa tag: nếu tag còn gắn bài viết, ta phải chặn — và chặn sao cho thông báo dễ hiểu.

api/src/controllers/tags.controller.ts
import type { Request, Response } from 'express';
import { AppError } from '../middleware/errorHandler.js';
import { prisma } from '../lib/prisma.js';
import { toSlug } from '../lib/slug.js';

const formatTag = (t: { id: number; name: string; slug: string; createdAt: Date }) =>
  ({ id: t.id, name: t.name, slug: t.slug, created_at: t.createdAt });

// T1 — GET /api/tags
export async function list(_req: Request, res: Response) {
  const tags = await prisma.tag.findMany({ orderBy: { name: 'asc' } });
  res.json(tags.map(formatTag));
}

// T2 — POST /api/tags
export async function create(req: Request, res: Response) {
  const { name } = req.body as { name?: string };
  if (!name?.trim()) throw new AppError(400, 'MISSING_FIELD', "Thiếu 'name'.");
  const tag = await prisma.tag.create({ data: { name, slug: toSlug(name) } });
  res.status(201).json(formatTag(tag));
}

// T3 — DELETE /api/tags/:id
export async function remove(req: Request, res: Response) {
  const id = parseInt(String(req.params.id), 10);
  if (isNaN(id)) throw new AppError(400, 'INVALID_TYPE', "'id' phải là số nguyên.");

  const tag = await prisma.tag.findUnique({
    where: { id },
    include: { _count: { select: { articles: true } } },
  });
  if (!tag) throw new AppError(404, 'TAG_NOT_FOUND', `Không tìm thấy tag id ${id}.`);

  // Kiểm TRƯỚC khi xóa để báo rõ số bài còn gắn (thay vì để DB ném P2003 khó hiểu)
  if (tag._count.articles > 0) {
    throw new AppError(409, 'TAG_IN_USE',
      `Tag còn gắn ${tag._count.articles} bài viết, không thể xóa.`);
  }
  await prisma.tag.delete({ where: { id } });
  res.status(204).send();
}
api/src/routes/tags.routes.ts
import { Router } from 'express';
import * as tags from '../controllers/tags.controller.js';

const router = Router();
router.get('/', tags.list);
router.post('/', tags.create);
router.delete('/:id', tags.remove);
export default router;
api/src/index.ts (thêm)
import tagsRouter from './routes/tags.routes.js';
app.use('/api/tags', tagsRouter);
Liên tưởng: Hai lớp bảo vệ cùng một quy tắc. Ở tầng database, ON DELETE RESTRICT đã chặn xóa tag đang dùng (DB sẽ ném lỗi P2003). Nhưng ta vẫn kiểm thêm ở application bằng _count.articles — không phải thừa, mà để trả về thông báo người-đọc-hiểu ("còn gắn 3 bài viết") thay vì lỗi DB khô khan. Đây là pattern phổ biến: DB đảm bảo toàn vẹn, application đảm bảo trải nghiệm.
