Chặng 3 · Backend + Cơ sở dữ liệu

Chạy PostgreSQL + áp schema + seed dữ liệu

Mục tiêu

Dựng PostgreSQL bằng Docker, áp migration tạo 3 bảng, và nạp dữ liệu mẫu để có cái mà thử.

Sản phẩm của bước: docker-compose.yml (service db) + api/prisma/seed.ts; DB có 3 bảng + 4 tag + 3 bài viết.

Sau bước này bạn sẽ thấy: Bạn sẽ có một database PostgreSQL chạy thật, đã đầy dữ liệu mẫu, sẵn sàng cho API truy vấn.

Hướng dẫn

Schema ở Chặng 2 mới là file .prisma — chưa có database thật. Bước này ta: (1) chạy PostgreSQL bằng Docker (không phải cài trực tiếp lên máy), (2) áp migration tạo bảng, (3) nạp dữ liệu mẫu để có cái mà thử ở các bước sau.

Tạo docker-compose.yml ở thư mục gốc dự án. Chú ý healthcheck — Docker dùng nó để biết DB đã sẵn sàng nhận kết nối chưa:

docker-compose.yml
services:
  # service api + web sẽ thêm ở Chặng 6
  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: miniwiki
      POSTGRES_USER: dev
      POSTGRES_PASSWORD: dev
    ports:
      - '5432:5432'
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ['CMD-SHELL', 'pg_isready -U dev -d miniwiki']
      interval: 5s
      timeout: 5s
      retries: 5

  pgdata:

Giờ viết api/prisma/seed.ts để nạp dữ liệu mẫu. Điểm đáng học ở đây là hàm slugify chuyển tiêu đề tiếng Việt thành slug ASCII (bỏ dấu, thay khoảng trắng bằng -):

api/prisma/seed.ts
import { PrismaClient } from '@prisma/client';
const prisma = new PrismaClient();

// Tiêu đề tiếng Việt -> slug ASCII (bỏ dấu, thay khoảng trắng = '-')
function slugify(text: string): string {
  return text
    .replace(/đ/g, 'd').replace(/Đ/g, 'D')
    .normalize('NFD').replace(/[̀-ͯ]/g, '') // bỏ dấu tổ hợp
    .toLowerCase().trim()
    .replace(/[^a-z0-9\s-]/g, '')
    .replace(/\s+/g, '-').replace(/-+/g, '-');
}

const TAGS = [
  { name: 'JavaScript', slug: 'javascript' },
  { name: 'Git',        slug: 'git' },
  { name: 'DevOps',     slug: 'devops' },
  { name: 'PostgreSQL', slug: 'postgresql' },
] as const;

// ARTICLES: 3 bài viết mẫu (title + content Markdown + tags).
// Xem nội dung đầy đủ ở commit đối chiếu bên dưới.
const ARTICLES = [ /* ... 3 bài, mỗi bài { title, content, publishedAt, tags } ... */ ];

async function main() {
  // 1. Upsert tags (slug là unique key -> chạy lại không tạo trùng)
  const tagMap = new Map<string, number>();
  for (const t of TAGS) {
    const tag = await prisma.tag.upsert({ where: { slug: t.slug }, update: {}, create: t });
    tagMap.set(t.slug, tag.id);
  }
  // 2. Upsert articles + gắn tag (xóa link cũ rồi tạo lại -> idempotent)
  for (const data of ARTICLES as any[]) {
    const slug = slugify(data.title);
    const article = await prisma.article.upsert({
      where: { slug },
      update: { title: data.title, content: data.content, publishedAt: data.publishedAt },
      create: { title: data.title, slug, content: data.content, publishedAt: data.publishedAt },
    });
    await prisma.articleTag.deleteMany({ where: { articleId: article.id } });
    for (const tagSlug of data.tags) {
      const tagId = tagMap.get(tagSlug);
      if (tagId) await prisma.articleTag.create({ data: { articleId: article.id, tagId } });
    }
  }
}

main()
  .catch((e) => { console.error('Seed thất bại:', e); process.exit(1); })
  .finally(() => prisma.$disconnect());
Nạp dữ liệu + lưu mốc bước 3.2
npm run seed                     # chạy prisma/seed.ts
# kiểm tra: docker exec -it <db> psql -U dev -d miniwiki -c "\dt"  -> 3 bảng
git add ../docker-compose.yml prisma/seed.ts
git commit -m "[step-3.2] Prisma client + docker-compose(db) + migrate + seed"
Liên tưởng: Vì sao dùng upsert thay vì create trong seed? upsert = "update nếu đã có, create nếu chưa". Nhờ vậy bạn chạy npm run seed nhiều lần mà không bị lỗi trùng slug hay nhân đôi dữ liệu — gọi là idempotent (chạy bao nhiêu lần kết quả vẫn như nhau). Một seed script tốt luôn nên idempotent.
Cạm bẫy: published_at phải nullable thật. Nếu ở bước trước bạn lỡ để migration DEFAULT NOW(), bài "nháp" (publishedAt = null) sẽ không tạo được đúng. Kiểm tra nhanh trong DB: cột published_at phải cho phép NULL.
