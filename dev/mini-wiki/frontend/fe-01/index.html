Chặng 4 · Frontend

Khởi tạo React + Vite + API client

Mục tiêu

Dựng khung front-end React 18 + Vite 6, định nghĩa kiểu dữ liệu và lớp gọi API có kiểu.

Sản phẩm của bước: Thư mục web/ chạy được; types.ts + lib/api.ts khớp hợp đồng API backend.

Sau bước này bạn sẽ thấy: Bạn sẽ mở http://localhost:5173 và thấy trang React chạy với hot-reload, sẵn sàng gọi backend.

Hướng dẫn

Backend đã chạy — giờ dựng front-end trong thư mục web/. Ta dùng Vite (build tool siêu nhanh, hot-reload ~50ms) + React 18 + react-router-dom để điều hướng. Điểm nhấn bước này là một lớp gọi API có kiểu (lib/api.ts) khớp đúng hợp đồng đã định ở backend.

Khai báo kiểu dữ liệu khớp với response backend. Chú ý ArticleListItem (danh sách, không content) khác Article (chi tiết, có content_html đã sanitize):

web/src/types.ts
export interface TagSummary { id: number; name: string; slug: string; }
export interface Tag extends TagSummary { created_at: string; }

// GET /api/articles — không có content để giảm payload
export interface ArticleListItem {
  id: number; title: string; slug: string;
  published_at: string | null;
  tags: TagSummary[];
}

// GET /api/articles/:slug — đầy đủ, content_html đã qua DOMPurify (an toàn)
export interface Article extends ArticleListItem {
  content: string;
  content_html: string;
  created_at: string;
  updated_at: string;
}

export interface Paginated<T> {
  data: T[];
  meta: { total: number; page: number; limit: number; query?: string };
}

Lớp gọi API — mỗi hàm có kiểu trả về rõ ràng, và đọc lỗi từ cấu trúc { error: { message } } của backend:

web/src/lib/api.ts
import type { Article, ArticleListItem, Paginated, Tag } from '../types';

const BASE_URL = import.meta.env.VITE_API_URL as string;

async function apiGet<T>(path: string): Promise<T> {
  const res = await fetch(`${BASE_URL}${path}`);
  if (!res.ok) {
    const body = (await res.json().catch(() => null)) as { error?: { message?: string } } | null;
    throw new Error(body?.error?.message ?? `HTTP ${res.status}`);
  }
  return res.json() as Promise<T>;
}

export function listArticles(params?: { tag?: string; page?: number }) {
  const qs = new URLSearchParams();
  if (params?.tag) qs.set('tag', params.tag);
  if (params?.page && params.page > 1) qs.set('page', String(params.page));
  const query = qs.toString();
  return apiGet<Paginated<ArticleListItem>>(`/articles${query ? `?${query}` : ''}`);
}
export const getArticle = (slug: string) =>
  apiGet<Article>(`/articles/${encodeURIComponent(slug)}`);
export const listTags = () => apiGet<Tag[]>('/tags');
export const searchArticles = (q: string, page?: number) => {
  const qs = new URLSearchParams({ q });
  if (page && page > 1) qs.set('page', String(page));
  return apiGet<Paginated<ArticleListItem>>(`/search?${qs.toString()}`);
};

Cạm bẫy: Đừng quên file src/vite-env.d.ts với nội dung /// <reference types="vite/client" />. File một dòng này khai báo kiểu cho import.meta.env — thiếu nó, lệnh npm run build (tsc -b && vite build) sẽ báo lỗi không nhận ra import.meta.env, dù npm run dev vẫn chạy bình thường. Một lần nữa: luôn build chứ không chỉ chạy dev.
